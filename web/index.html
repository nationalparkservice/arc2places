arc2places

Manage Places data with ArcGIS

About arc2places

arc2places is a suite of tools for uploading National Park Service ArcGIS data to the NPS Places system. It was built to support the centennial data call.

arc2places provides an ArcGIS Toolbox interface for using these tools. This document describes how to install and use the arc2places toolbox. Source code and additional information on the arc2places tools can be found on GitHub.

Installing

Install Prerequisites

arc2places requires an extension to Python not provided by the standard ArcGIS Python installation. This extension handles the authentication with the Places server. Installation of the extension requires a tool that is also not provided by ESRI. To complicate this further, ESRI puts python in a different location on each release, and they also provide multiple versions of python with each release. The installaller provided is fairly simple and may not work in all cases. If you run into trouble, please contact us for installation support.

Assumptions

  • You have ArcGIS 10.1sp1 or greater installed on your computer with python installed in the default location.
  • You have administrative permissions on your computer (at least write permission to the python folder).
  • Windows knows how to open python files (*.py) with the Python interpreter ArcGIS will use (true for standard ArcGIS install).
  • You do not have 64bit background geoprocessing enabled. It can be installed, but if it is enabled, then you will need to repeat the prerequisite installation for both the 32bit and 64bit versions of Python.

Instructions

  1. Download the arc2places pre-installer.zip (if clicking the link does not download a zip file, then right click on the link and select 'Save link as...').
  2. Extract/unzip pre-installer.zip to a temporary location that is easily accessible.
  3. Use windows explorer to browse to the extraction location.
  4. If you are feeling lucky
    • Double click on install.py in the extraction location.
  5. Otherwise:
    1. Right click on install.py and select Edit with IDLE.
    2. In the IDLE Editor, select Run -> Run Module from the menu (or press F5).
    3. This should open another window that runs the installation program and displays diagnostic output.
    4. If you see error messages in the IDLE window, contact us for installation support.
    5. Close both IDLE windows.
  6. Proceed to instruction for Install/Update Toolbox.
  7. If everything works, you can delete pre-installer.zip and the extraction location.

Install/Update Toolbox

The arc2places tool and the translators it uses are still changing quite a bit. Please make sure you are using the most recent version, and check back here for updates before doing a large data upload.
  1. Download arc2places.zip (updated 2015-10-02).
  2. Extract/unzip arc2places.zip to a local folder that is accessible to ArcCatalog.
  3. If you are updating, you should unzip 'on top of' and replace files in the existing installation. You will need to stop and restart ArcCatalog to use the new version. Do not try running ArcCatalog with two versions of arc2places installed in two different locations, it just gets confused.
  4. Open ArcCatalog and browse to the extraction location of arc2places.
  5. Try and expand the arc2places toolbox
    • If it opens and you see the contents of the toolbox, installation was probably successful. (It may still fail when executing the tool if you have 64bit background geoprocessing enabled. See the installation prerequisites for more information.)
    • If there is a red x on the toolbox icon, or it does not expand, there was a most likely a problem installing the prerequisites. Please try again or contact us for installation support.
  6. It is recommended that you turn off background geoprocessing while first using arc2places so that you can see any diagnostice warnings or errors.

Using

The toolbox can be used in ArcCatalog and ArcMap. The documentation below is based on ArcCatalog. The toolbox has only been tested on ArcGIS 10.3.x It should work on ArcGIS 10.1 sp1 or higher. It does not yet work with ArcGIS Pro.

Testing

Before uploading any production data, you are encouraged to test with small datasets. Testing should be done with the test version of the Places database. The tools have a checkbox to select publishing to test vs. production. View the test version of Places at http://insidemaps.nps.gov/dev/index.html. You should verify that sensitive features were not sent, and that the items got the correct tags. You can also check the symbology and labels, however features may appear different when rendered in Park Tiles.

During testing you are encouraged to use the tools under 'Features to Places (Step by Step)' toolkit. If there are validation problems, the tools in the 'Useful Tools' toolkit may be useful. Once you are comfortable with the process, and the fitness of your data, you can use 'All in One Tools/Features to Places' to eliminate some of the steps.

Features to Places

Send the public features in your ArcGIS dataset to Places.

Translators are provided for:

  • Buildings (footprints)
  • Points of Interest
  • Parking Lots (footprints)
  • Trails
  • Roads

The translators assume the data is largely compliant with the eGIS data standards, however they make a good attempt to support reasonable variants. It is possible (and even pretty easy) to modify the translators to support other datasets or non-standard data.
